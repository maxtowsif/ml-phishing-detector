üß† 3. Machine Learning Methodology

üìä Dataset and Training

Training Dataset: Over 10,000 labeled phishing and legitimate URLs from comprehensive threat intelligence sources.

üî¨ Feature Engineering Process

  1. Lexical Features (13 features):
    • URL length, hostname length
    • Special character counts (dots, hyphens, @, ?, &, |, /)
    • Digit ratio and subdomain presence
  2. Security Features (8 features):
    • HTTPS presence and SSL certificate validation
    • Suspicious keyword detection
    • Domain age and IP address usage
    • Path length and query parameter analysis
  3. Domain Features (9 features):
    • Top-level domain analysis
    • Vowel/consonant ratios
    • Character entropy calculation
    • Domain reputation indicators

üîß Feature Selection Process

üîå 10. API Reference

üß† Core Analysis Functions

model.predict_phishing(url)

Parameters: - url (str): URL to analyze Returns: - dict: { 'prediction': 'Phishing' or 'Legitimate', 'confidence': float (0-1), 'risk_score': float (0-100), 'risk_level': 'Very Low' | 'Low' | 'Medium' | 'High', 'model_used': 'Random Forest' }

features.extract_features(url)

Parameters: - url (str): URL for feature extraction Returns: - dict: { 'length_url': int, 'nb_dots': int, 'https': bool, 'ssl_valid': bool, 'suspicious_keywords': int, 'feature_importance': dict, ... (30+ features) }

url_analysis.analyze_url_security(url)

Parameters: - url (str): URL for security analysis Returns: - dict: { 'ssl_info': {...}, 'dns_info': {...}, 'whois_info': {...}, 'http_info': {...}, 'security_score': float (0-100), 'risk_factors': list, 'recommendations': list }

üîê Authentication Functions

auth.check_permission(permission)

Parameters: - permission (str): Required permission Returns: - bool: True if user has permission Permissions: - 'detection': Basic URL analysis - 'bulk': Bulk analysis capability - 'dashboard': Analytics dashboard access - 'admin': Administrative functions
